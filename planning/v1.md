V1 Milestone
============

Goal: A fully functional end-to-end writing experience in the web app, covering the free tier feature set. No paid features, no Electron desktop app.


In Scope
--------

Project
- Create a new project
- Open an existing project from local filesystem (via recent projects list)
- Rename and delete a project

Sections
- Create, rename, delete sections
- Reorder sections via drag and drop
- Sections can be grouped (e.g. a chapter containing scenes) — one level of nesting only in v1
- Groups can be collapsed/expanded in the sections sidebar
- Each section stored as a flat .md file in the sections/ directory
- Order and grouping managed by a manifest in project.json (not by filenames or folder structure)

Editor
- Clean rich text editor (bold, italic, headings, paragraphs)
- Search and replace within a section

Import
- Open a .txt or .md file and add it as a section

Reference
- Generic container: each item has a type, a name, and freeform fields
- Built-in types for v1: Character, Location, Timeline, Scenes, Notes, Research
- Types defined in reference/collections.json; items stored as individual JSON files in reference/
- Create, edit, delete reference items
- Board view (freeform canvas with nodes, edges, rectangles, annotations)
- Grid view (cards by type)
- Sidebar with manifest-driven ordering, groups, and drag-and-drop

Word & Page Count
- Per-section: word count and character count displayed in a status bar at the bottom of the editor
- Per-project: total word count and estimated page count (at 250 words/page) shown in the header
- Counts are calculated from content on the fly, nothing stored

Writing Goals
- User can set word count goals: per session, daily, weekly, or monthly
- Session progress tracked live as the user writes (words written since session opened)
- Daily/weekly/monthly progress tracked via a writing activity log
- Net words written per day calculated by diffing word count at session start and end
- Data stored in writing-log.json in the project root:
    {
      "goals": { "daily": 1000, "weekly": 5000 },
      "log": [ { "date": "2026-02-16", "words": 847 }, ... ]
    }
- Stats & Goals accessible via a clickable word count indicator in the header
  that expands into a full goals/progress view

Export
- Plain text: export full project or individual section as .txt
- PDF: browser print-to-PDF with a custom print stylesheet


Out of Scope for V1
-------------------
- Firebase Auth / user accounts (moved to v2)
- AI chat sidebar (moved to v2)

Project File Structure
----------------------

my-project/
  project.json              # title, author, created date, settings, sections manifest
  writing-log.json          # writing goals and daily activity log
  sections/                 # flat directory of content files
    scene-one.md
    scene-two.md
    interlude.md
  metadata/
    collections.json        # type definitions and field templates for built-in types
    john-smith.json         # { "id": "...", "type": "character", "name": "John Smith", "fields": { ... } }
    the-old-house.json      # { "id": "...", "type": "location", ... }
  assets/                   # images and reference files

Sections manifest (inside project.json):
  "sections": [
    {
      "id": "part-one",
      "title": "Part One",
      "type": "group",
      "children": [
        { "id": "scene-one", "title": "The Arrival", "file": "scene-one.md" },
        { "id": "scene-two", "title": "First Morning", "file": "scene-two.md" }
      ]
    },
    { "id": "interlude", "title": "Interlude", "type": "section", "file": "interlude.md" }
  ]


UI Structure
------------

Screens:

  Home (no dashboard)
    - App opens to a simple start screen
    - New Project button
    - Open Project action: shows list of recent projects + option to browse filesystem
  Editor View
    - Header: [≡ sections toggle]  Project / Section title  [Reference]  [⋯ menu]
    - Left sidebar (sections list): collapsible
        - Open by default for new projects, closed for returning projects
    - Center: editor (expands to full width when sidebars collapsed)
    - [Reference] button in header navigates to Reference View
    - [⋯] menu: export, import, rename project, project settings

  Reference View (full-screen, separate from editor)
    - Header: [← Back to Editor]  Reference  [Board | Grid]
    - Left sidebar: collection types with items, groups, DnD reordering
    - Main area: Board (freeform ReactFlow canvas) or Grid (card layout by type)

  Settings
    - Project settings (title, subtitle)
    - Editor settings (font, size, paper mode, spell check, words per page)


Build Order
-----------

Suggested sequence to reach a usable state as early as possible:

1. Monorepo scaffolding (Turborepo, shared packages)
2. Home screen — new project, recent projects list
3. Project creation and local file save/load
4. Section management (CRUD + reorder + grouping)
5. Rich text editor
6. Word & page count (status bar + project header)
7. Writing goals (writing-log.json, stats & goals panel)
8. Reference view (board, grid, sidebar with manifest-driven ordering)
9. Import (.txt / .md)
10. Export (plain text + PDF + standard manuscript)
11. UI polish: empty states, visual consistency, error handling
